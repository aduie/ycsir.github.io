Android 中 Flow vs LiveData 实战对比与最佳实践

羽晨sir 💕
  2026-01-18 13:41:18 2026-01-18 13:41:18 Created   2026-03-18 13:42:55 2026-03-18 13:42:55 Updated  
  • 学习 
 

一、背景

在 Android 开发中，数据驱动 UI 已经成为主流模式。早期我们主要使用 LiveData，而现在 Kotlin Flow 越来越成为主流选择。

本文结合实际项目经验，对 Flow 和 LiveData 进行对比，并给出推荐使用方式。

二、核心区别

对比点 LiveData Flow
生命周期感知 ✅ 自动感知 ❌ 需手动处理
Kotlin支持 一般 ✅ 原生支持
异步能力 较弱 ✅ 强
操作符 少 丰富（map/flatMap）
冷流/热流 热流 冷流

三、实际使用场景

1. UI 层（推荐 LiveData）

👉 优点：

  • 自动绑定生命周期
  • 不容易内存泄漏

2. 数据层（推荐 Flow）

👉 优点：

  • 支持链式调用
  • 更适合复杂数据流

四、项目中的最佳实践（重点）

在我的项目中，采用：

👉 Flow + LiveData 混合模式

👉 这样可以做到：

  • 数据层：Flow（灵活）
  • UI层：LiveData（安全）

五、总结

  • UI层推荐：LiveData
  • 数据层推荐：Flow
  • 最佳实践：Flow 转 LiveData

六、个人思考

Flow 在复杂业务中明显优于 LiveData，但在 UI 层直接使用 Flow 会增加复杂度，因此混合模式是目前最优解。
