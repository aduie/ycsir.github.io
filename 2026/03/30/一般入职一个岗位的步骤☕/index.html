一般入职一个岗位的步骤☕

羽晨sir 💕
  2026-03-30 15:44:52 2026-03-30 15:44:52 Created   2026-03-30 16:29:58 2026-03-30 16:29:58 Updated

说明：本文档针对新手定制，所有操作步骤清晰可落地，适配公司规模（小型公司），重点覆盖「入职办理」「开发环境搭建」全流程，每个步骤标注具体操作、注意事项，避免踩坑。

一、入职办理

核心目标：完成入职手续，领取办公物资，熟悉基础对接人

  1. 到公司后，直接前往前台，告知前台“我是今天入职XX岗位的XXX”，出示身份证原件。

  2. 前台引导到HR工位，领取入职相关表格（入职登记表、劳动合同、保密协议），按要求填写：

    1️⃣入职登记表：如实填写个人信息（姓名、电话、紧急联系人、学历、工作经历等），字迹清晰。

    2️⃣劳动合同：仔细查看薪资（8-13K，确认发放时间）、试用期、岗位职责等条款，确认无误后签字（一式两份，自己留存一份）。

    3️⃣保密协议：重点看“公司代码、项目文档、接口信息不得外泄”相关条款，签字确认。

  3. HR开通企业沟通工具（企业微信/飞书/钉钉），告知账号密码（一般为手机号，初始密码可修改），添加直属leader、团队成员、HR好友。

  4. HR或直属leader带你到指定工位，介绍工位周边（打印机、茶水间、卫生间）位置，介绍团队成员（简单打招呼即可，不用刻意社交）。

注意：所有签字文件仔细核对，有疑问及时问HR，不要随意签字；牢记直属leader姓名和工位，后续有问题优先找他。

二、开发环境搭建

核心目标：完成所有开发相关软件安装、权限开通，成功拉取项目代码并编译运行，打通测试环境，确保能正常开展后续开发工作（新手重点看，每一步都要操作到位）

第一步：开通项目Git仓库权限（关键第一步）

  1. 找到直属leader，说明“需要Git仓库地址和权限，用于拉取项目代码”。
  2. leader会提供Git仓库地址（示例：https://gitee.com/tumingkeji/android-project.git ，实际以公司提供的为准），同时告知你的Git账号（一般为公司邮箱）和初始密码。
  3. leader操作开通你的仓库权限（读写权限，确保能拉取、提交代码），开通后会通知你。
  4. 测试权限：打开浏览器，输入Git仓库地址，输入账号密码，能成功登录、看到项目目录，即为权限开通成功。

注意：不要泄露Git账号密码、仓库地址，不要随意删除仓库内的文件/分支。

第二步：安装必备开发软件（按顺序安装，避免出错）

所有软件优先安装公司提供的安装包（HR或leader会发），没有则按以下步骤下载安装，全程默认下一步即可，特殊步骤标注说明。

1. 安装JDK（Android开发必备，版本按公司要求，一般为JDK11）

  1. 获取JDK安装包（leader提供，或从Oracle官网下载，注意选择对应系统：Windows/macOS）。
  2. 双击安装包，点击“下一步”，选择安装路径（建议默认路径，方便后续配置，如C:\Program Files\Java\jdk-11）。
  3. 配置环境变量（新手重点，一步都不能漏）：
    1. Windows系统：右键“此电脑”→“属性”→“高级系统设置”→“环境变量”。
    2. 在“系统变量”中，点击“新建”，变量名：JAVA_HOME，变量值：JDK安装路径（如C:\Program Files\Java\jdk-11），点击确定。
    3. 找到“系统变量”中的“Path”，双击打开，点击“新建”，输入“%JAVA_HOME%\bin”，点击确定，再点击“新建”，输入“%JAVA_HOME%\jre\bin”，点击确定。
    4. 测试是否安装成功：打开电脑“命令提示符”（Win+R，输入cmd），输入“java -version”，按下回车，能显示JDK版本号（如java version “11.0.18”），即为安装成功。

2. 安装Git（用于拉取、提交代码）

  1. 获取Git安装包（leader提供，或从Git官网下载），双击安装。
  2. 安装过程中，所有步骤默认下一步，唯一注意：在“Select Components”步骤，勾选“Git Bash Here”“Git GUI Here”（方便后续右键快速打开Git命令行）。
  3. 测试是否安装成功：右键空白处，选择“Git Bash Here”，输入“git –version”，按下回车，显示Git版本号（如git version 2.42.0.windows.1），即为安装成功。
  4. 配置Git（关联公司账号）：在Git Bash中，依次输入以下两条命令，每输完一条按回车（替换成自己的信息）：
    1. git config –global user.name “你的姓名（公司登记的）”
    2. git config –global user.email “你的公司邮箱（Git账号）”

3. 安装Android Studio（核心开发工具）

  1. 获取Android Studio安装包（leader提供，避免自己下载版本不兼容），双击安装。
  2. 安装步骤：
    1. 点击“Next”，勾选“Android Studio”“Android SDK”“Android Virtual Device”（三个都要勾选，缺一不可），点击Next。
    2. 选择安装路径（建议默认，或安装到D盘，避免C盘空间不足），点击Next。
    3. 点击“Install”，等待安装完成（约5-10分钟，看电脑配置）。
    4. 安装完成后，打开Android Studio，首次打开会提示“Configure”，选择“SDK Manager”。
    5. 在SDK Manager中，勾选公司项目对应的SDK版本（leader会告知，一般为Android 12/13），勾选“SDK Build-Tools”（选择与SDK版本匹配的版本），点击“Apply”，等待下载安装（约10-20分钟，需耐心等待）。
  3. 测试是否安装成功：打开Android Studio，能正常进入主界面，SDK下载完成后，无报错提示，即为安装成功。

注意：Android Studio下载SDK时，若出现网络超时，告知leader，可能需要配置公司内网镜像。

Android Studio 内网镜像 / 代理配置（解决 SDK 下载超时）:通过配置公司内网代理或镜像源，让下载走公司内部通道，从而极速成功。

1️⃣代理地址 (Proxy Host):

例如 http://proxy.tumingkeji.com 或 192.168.1.50

代理端口 (Proxy Port): 例如 8080、3128

是否需要账号密码 (Authentication): 大部分公司需要，问清楚账号密码

2️⃣全局 Gradle 代理配置（最推荐，一劳永逸）

👉打开 Android Studio。

👉点击顶部菜单栏 File → Settings (Windows/Linux) 或 Android Studio → Preferences (Mac)。

👉在设置窗口左侧，找到 Appearance & Behavior → System Settings → HTTP Proxy。

👉选择 Manual proxy configuration (手动配置代理)。

填写信息：

​ HTTP proxy: 输入刚才问到的 代理地址 (如 192.168.1.50)

  • ​ Port: 输入 代理端口 (如 8080)

👉关键步骤：勾选 Use proxy for all protocols (为所有协议使用代理)。

👉如果公司代理需要账号密码，勾选 Proxy requires authentication，并输入你的 Login (账号) 和 Password (密码)。

👉点击底部 OK 保存。

此时 Android Studio 会提示重启，点击 Restart 重启 IDE。

第三步：拉取项目代码（新手最容易出错，仔细看步骤）

  1. 在电脑桌面，右键空白处，选择“Git Bash Here”，打开Git命令行。
  2. 输入命令“cd 桌面”（按下回车，切换到桌面目录，方便后续查找项目文件夹）。
  3. 输入拉取代码命令：git clone 公司提供的Git仓库地址（示例：git clone https://gitee.com/tumingkeji/android-project.git），按下回车。
  4. 此时会提示输入Git账号和密码（就是leader给的账号，输入时密码不显示，正常输入即可，输完按回车）。
  5. 等待代码拉取完成（时间看项目大小，一般5-10分钟，命令行显示“done”即为完成）。
  6. 拉取完成后，桌面会出现一个与项目名称一致的文件夹（如android-project），即为项目代码文件夹。

常见问题解决：

  • 提示“权限不足”：联系leader，确认Git仓库权限已开通，重新输入账号密码尝试。
  • 拉取超时：检查网络，若为公司内网，告知leader，确认内网能访问Git仓库。

第四步：尝试编译运行项目（核心验证步骤）

  1. 打开Android Studio，点击“Open”，找到桌面的项目文件夹（android-project），点击“OK”，导入项目。
  2. 导入后，Android Studio会自动同步项目（右下角会显示“Syncing”），同步过程中可能会提示“缺少依赖”“SDK版本不匹配”，按以下方式解决：
    1. 缺少依赖：点击提示框中的“Install Missing Dependencies”，自动下载安装，等待完成。
    2. SDK版本不匹配：点击提示框中的“Sync Project with Gradle Files”，或打开SDK Manager，安装对应版本的SDK。
  3. 同步完成后，配置模拟器（或连接真机）：
    1. 模拟器：点击Android Studio顶部的“AVD Manager”（手机图标），点击“Create Virtual Device”，选择一个手机型号（如Pixel 5），选择已下载的SDK版本，点击“Finish”，创建模拟器。
    2. 真机：用数据线连接电脑和自己的安卓手机，在手机上开启“开发者模式”（设置→关于手机→连续点击版本号7次），开启“USB调试”（开发者选项中勾选），电脑会自动安装手机驱动，驱动安装完成后，Android Studio会识别到手机。
  4. 编译运行：点击Android Studio顶部的“Run”按钮（绿色三角形），选择模拟器/真机，点击“OK”，开始编译运行项目。
  5. 验证：若模拟器/真机上能成功打开App，无崩溃、无报错，即为编译运行成功；若报错，截图保存，及时找leader或资深同事帮忙解决（新手报错很正常，不要硬憋）。

第五步：配置内网、VPN（若公司需要，按以下步骤操作）

  1. 找到HR或行政，领取公司内网配置说明、VPN安装包（初创公司可能不需要VPN，以公司要求为准）。

  2. 配置内网：

    1. Windows系统：右键“网络”→“属性”→“更改适配器设置”，右键“以太网”→“属性”，找到“Internet协议版本4（TCP/IPv4）”，双击打开。
    2. 选择“使用下面的IP地址”“使用下面的DNS服务器地址”，按公司提供的内网IP、子网掩码、网关、DNS填写，点击确定，即可连接内网。

  3. 安装配置VPN（若有）：

    1. 双击VPN安装包，默认下一步安装，安装完成后，打开VPN客户端。
    2. 输入公司提供的VPN服务器地址、账号、密码，点击“连接”，连接成功后，即可访问公司内网资源。

  4. 测试：连接内网/VPN后，尝试打开公司内网地址（如接口文档地址），能正常打开即为配置成功。

第六步：配置接口文档、测试服地址（开发必备，避免后续踩坑）

  1. 找leader获取接口文档地址（一般为在线文档，如Swagger、ShowDoc，示例：http://192.168.1.100:8080/swagger-ui.html）、测试服地址（示例：http://test.tumingkeji.com:8081）。

  2. 打开浏览器，输入接口文档地址，能正常打开、查看接口（请求方式、参数、返回值），即为接口文档可访问。

  3. 配置测试服地址到项目中：

    1. 打开Android Studio中的项目，找到“res”→“values”→“strings.xml”文件（或项目中专门配置接口地址的文件，leader会告知）。
    2. 找到“base_url”（基础地址），将其值修改为公司提供的测试服地址，保存文件。
    3. 重新编译项目，确保修改后无报错，测试服地址配置完成。

  4. 验证：在App中尝试点击需要调用接口的功能（如登录、首页加载），若能正常获取数据（无“接口请求失败”提示），即为测试服配置成功。

注意：接口文档要收藏到浏览器，后续开发时随时查看；测试服地址不要修改为正式服地址，避免影响正式环境。

三、环境搭建完成后，必做的3件事

  1. 再次确认：项目能正常编译运行、接口能正常调用、内网/VPN能正常连接，无报错。
  2. 整理笔记：把Git仓库地址、账号密码（记在自己的笔记本上，不要存在电脑桌面）、接口文档地址、测试服地址、leader联系方式，统一记录好，方便后续查阅。
  3. 向leader汇报：找到leader，说“报告XX（leader姓名），我的开发环境已经搭建完成，项目能正常编译运行，接口和测试服也已配置好”，让leader放心。

四、新手注意事项（重点提醒）

  • 所有操作若不确定，先问leader或资深同事，不要自己瞎操作（尤其是修改项目配置、删除代码）。
  • 软件安装、代码拉取、编译过程中，遇到报错不要慌，截图保存，详细说明问题，找同事帮忙解决（新手第一天报错很正常，没人会笑话你）。
  • 不要随意修改项目中的核心代码、配置文件，若需要修改，先告知leader，得到允许后再操作。
  • 全程保持耐心，环境搭建可能会遇到各种小问题，慢慢解决，确保每一步都落实到位，为后续开发打下基础。
