🚀 一、先搞清楚：测试工具开发是干嘛的？

简单说一句人话：

👉 不是做 App 给用户用，而是做工具给“开发 / 测试人员用”

常见场景比如：

  • 一键切换环境（测试 / 生产）
  • 查看接口请求日志
  • 模拟接口返回数据
  • 自动点击 / 自动化测试
  • 抓包 / WebSocket 调试
  • 崩溃日志收集

🧠 二、记录要做的第一个工具

✅【接口调试工具 App】（类似 Postman 的简化版）

功能非常简单：

  • 输入 URL
  • 选择 GET / POST
  • 点击发送请求
  • 展示返回结果

👉 这个东西实用性强，因为：

  • 涉及网络请求
  • UI
  • JSON处理
